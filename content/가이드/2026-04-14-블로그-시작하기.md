## 글을 어디에 두나요?

모든 글은 **`content` 아래 주제 폴더**에 둡니다. 주제 이름은 URL 경로에 그대로 쓰입니다.

```text
content/
  가이드/
    2026-04-14-블로그-시작하기.md
  개발/
    2026-04-14-next-js-팁.md
```

## 파일 이름 규칙

반드시 **`YYYY-MM-DD-제목슬러그.md`** 형식이어야 합니다.

- 날짜: ISO 형식 `2026-04-14`
- 그 다음 하이픈 뒤에 영문·한글 슬러그 모두 가능합니다.

## 프론트매터 (선택)

파일 맨 위에 YAML 블록을 넣을 수 있습니다.

```yaml
---
title: 화면에 보일 제목
description: 목록·메타 설명
date: 2026-04-14
---
```

`title`이 없으면 파일명의 슬러그 부분을 읽기 좋게 바꿔 씁니다. `date`를 생략하면 파일명의 날짜를 사용합니다.

## 마크다운 예시

**굵게**, *기울임*, [링크](https://nextjs.org), 인라인 `코드`를 쓸 수 있습니다.

### 목록

1. GitHub에 저장소를 푸시합니다.
2. Vercel에서 해당 저장소를 import 합니다.
3. Framework Preset이 Next.js인지 확인한 뒤 배포합니다.

### 표

| 단계 | 설명 |
|------|------|
| 빌드 | `npm run build` |
| 로컬 | `npm run dev` |

### 코드 블록

```ts
export function hello(name: string) {
  return `Hello, ${name}`;
}
```

인용문도 지원합니다.

> 디자인은 프로젝트 루트의 `Design.md`를 기준으로 적용되어 있습니다.
